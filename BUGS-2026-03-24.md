### ~~BUG-13: Client Portal Shows "0 article" on Order (P2)~~ ✅ FIXED

**Status:** Fixed — portal lookup API switched from anon key to service role client.

**Root cause:** The portal lookup API used `createClient(url, anonKey)` — the `anon` role had no GRANT permissions on the `garment` table, so the embedded `garment(id)` select silently returned nothing, making the count always 0.

**What was fixed:**
- Switched to `createServiceRoleClient()` (same as the status lookup route)
- Added `OrderWithGarments` interface for type safety
- Files changed: `src/app/api/portal/lookup/route.ts`
