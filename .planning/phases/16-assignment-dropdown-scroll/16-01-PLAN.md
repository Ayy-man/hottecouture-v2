# Phase 16 Plan 01: Fix Assignment Dropdown Viewport Collision

<objective>
Fix the custom Select component's dropdown positioning so it opens upward when there isn't enough space below the trigger. Currently, SelectContent always positions below the trigger without checking viewport bounds, causing items to be cut off at the bottom of the screen.
</objective>

## Root Cause

The issue is in `src/components/ui/select.tsx` lines 113-122. The `SelectContent` component calculates position as `rect.bottom + window.scrollY + 4` (always below trigger) without checking if the dropdown would extend beyond the viewport.

All 5 assignment dropdowns in the app use this same base component, so fixing it once fixes all instances:
- garment-services-step.tsx (per-service assignment)
- assignment-step.tsx (bulk + per-item assignment)
- task-management-modal.tsx (stage + assignee dropdowns)
