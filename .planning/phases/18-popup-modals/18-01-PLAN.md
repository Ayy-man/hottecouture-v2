# Phase 18 Plan 01: Popup Modals for Workload View + Modal UX Improvements

<objective>
Make order detail viewing work as an inline popup modal on the workload page instead of navigating away to the board. Also add ESC key and backdrop click to close the existing OrderDetailModal.
</objective>

## Research Summary

**Current state:**
- **Kanban board**: Already uses `OrderDetailModal` as a fixed overlay — clicking an order card opens modal inline. No changes needed.
- **Workload page**: Uses `<Link href="/board?order=...">` which navigates away from the workload page to the board, then auto-opens the modal there. User loses their place.
- **OrderDetailModal**: Fixed overlay with close button, but missing ESC key handler and backdrop click-to-close.

**What needs to change:**
1. Workload page: Import `OrderDetailModal`, add state for selected order, replace `<Link>` with click handler that opens the modal inline
2. OrderDetailModal: Add ESC key handler and backdrop click handler
