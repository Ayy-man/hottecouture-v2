# Phase 13 Plan 01: Fix Price Editing Regression in Services Table

<objective>
Client reports that price editing in the services/alterations table was previously working but is now broken (regression). Re-enable the ability to modify service prices from the table view, and add a three-dot menu with Modify/Export/Delete options.
</objective>

## Context

- Audrey quote: "Avant on etait rendu que quand que c'etait un tableau, on pouvait modifier le prix"
- OTO response: "Je pense que j'ai pas active cette fonction la"
- This is a REGRESSION - feature was working, now broken
- Likely disabled during Phase 2 (item-level pricing) refactoring

## Tasks

### Task 1: Locate the Services Table Component
- Find the services/alterations management table (likely in admin section)
- Check `src/app/admin/` or `src/app/settings/` for service management pages
- Identify the table component rendering services with prices

### Task 2: Identify What Broke
- Check git history for changes to the services table component
- Look for disabled edit functionality, removed onClick handlers, or conditional rendering that hides edit controls
- Check if the API endpoint for updating service prices still exists

### Task 3: Re-enable Price Editing
- Restore inline editing or add edit modal for service prices
- Ensure the PATCH/PUT endpoint for service updates works
- Fields that should be editable: service name, price, estimated time

### Task 4: Add Three-Dot Context Menu
- Add a three-dot (ellipsis) menu to each row in the services table
- Menu options:
  - **Modify** - Opens edit modal/inline editing for the service
  - **Export** - Exports this service's data (or the full list)
  - **Delete** - Removes the service (with confirmation dialog)
- Use existing Radix UI dropdown menu component

### Task 5: Test Price Editing Flow
- Edit a service price -> verify it saves to database
- Edit service name -> verify it saves
- Delete a service -> verify confirmation dialog appears -> verify deletion
- Verify changes reflect immediately in the intake form service selection

## Key Files
- `src/app/admin/` or `src/app/settings/` (service management page)
- `src/app/api/` (service CRUD endpoints)
- `src/components/ui/dropdown-menu.tsx` (Radix dropdown)

## Acceptance Criteria
- [ ] Can click to edit service price in table view
- [ ] Price changes save to database immediately
- [ ] Three-dot menu visible on each service row
- [ ] Modify option opens editing interface
- [ ] Delete option shows confirmation dialog before deleting
- [ ] No regressions to intake form service selection

`fix: restore price editing in services table and add context menu`
