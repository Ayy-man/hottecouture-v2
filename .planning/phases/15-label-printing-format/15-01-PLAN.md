# Phase 15 Plan 01: Fix Label Printing Format for Individual Label Printer

<objective>
Change the label output from a full-page grid layout (2 labels on letter paper) to individual labels sized for Audrey's label printer. Each label should be one "page" so the label printer receives one label at a time.
</objective>

## Research Summary

**Current Implementation:**
- `/labels/[orderId]/page.tsx` — Client component that fetches order data, generates QR codes, renders labels in a 2-column CSS grid on letter paper
- Print CSS: `@page { size: letter; margin: 0.5in; }` with labels at ~45vh each (2 per page)
- `LABEL_CONFIG.copyCount = 2` — prints 2 copies per garment
- Each label contains: QR code, order #, garment type, label code, client name, rush indicator, status, due date
- Download-as-PNG feature also exists (canvas-based)

**Problem:**
Client has a dedicated label printer ("une machine à 4"). Current output targets letter-size paper with multi-label layout. Label printer expects individual labels, one at a time.

**Solution approach:**
Use CSS `@page` with the label printer's physical dimensions. Each label = one CSS page. When sent to the label printer, each page prints on one physical label. This is the standard web-printing approach for label printers — no special protocols needed.

## Tasks

### Task 1: Add label printer dimensions to config

**File:** `src/lib/config/production.ts`

Add label printer dimensions to `LABEL_CONFIG`:

```typescript
export const LABEL_CONFIG = {
  copyCount: 2,
  showCopyIndicator: true,
  copyIndicatorFormat: (current: number, total: number) => `${current} de ${total}`,
  // Label printer paper size (4" x 2" is standard for most label printers)
  labelWidth: '4in',
  labelHeight: '2in',
};
```

**Note:** 4" × 2" is the most common individual label size for Dymo/Brother-style printers. If the client's printer uses different dimensions, this is the one place to change.

### Task 2: Rewrite labels page for individual label printing

**File:** `src/app/labels/[orderId]/page.tsx`

Replace the current 2-column grid layout with a vertical stack where each label is in its own container with `break-after: page`.

**Changes to the JSX:**
1. Remove `grid grid-cols-2 gap-6` wrapper
2. Wrap each label in a container with `break-after: page` (except the last one)
3. Size each label to fill the label printer's page area
4. Keep the same label content (QR, order #, type, code, client, rush, status, due date)
5. Optimize layout for the smaller label dimensions — horizontal layout instead of vertical stack

**New label layout (horizontal, fits 4" × 2"):**
```
┌──────────────────────────────────┐
│ [QR]  #123        garment type   │
│ [QR]  Client Name   label_code   │
│ [QR]  RUSH    Status  Échéance   │
│       copy 1 de 2                │
└──────────────────────────────────┘
```

Left side: QR code (~1.2" square)
Right side: All text info stacked

**Changes to print CSS (inline `<style jsx global>`):**
```css
@media print {
  @page {
    size: 4in 2in;
    margin: 0;
  }
  /* ... rest of print hiding */
}
```

### Task 3: Update downloadAsImage for new layout

The `downloadAsImage()` canvas function currently renders a 2-column grid. Update to render labels in a single column strip (one label after another vertically), matching the new print layout. Each canvas label should use horizontal layout matching the print version.

### Task 4: Verify screen preview looks reasonable

The screen preview (non-print) should show labels stacked vertically with a visual separator between them, so the user can see exactly what will print. Each label preview should be rendered at the configured dimensions with a border.
