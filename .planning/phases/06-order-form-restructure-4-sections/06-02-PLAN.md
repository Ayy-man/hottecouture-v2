**Core logic:**
1. Fetch services from Supabase filtered to `category = 'accessories'` (or 'accessory' to handle singular/plural).
2. Display available accessory services in a flat list (no tabs, no garment type selection).
3. Each service row shows: name, price, quantity input (decimal: `type="number" step="0.25" min="0.25" inputMode="decimal"`), add button.
4. When user adds an accessory service:
   - If `data` (garments from AlterationStep) is NOT empty: add the accessory service to the LAST garment's services array.
   - If `data` IS empty (no garments from alteration): auto-create a garment with `type: 'Accessoires'`, `garment_type_id: null`, `labelCode: nanoid(8).toUpperCase()`, `services: []`, then add the accessory service to it.
   - The service object MUST have `isAccessory: true` and `estimatedMinutes: 0` (accessories have no time).
5. Show a summary of added accessories at the bottom with qty, price, and remove button.
6. "Suivant" button always enabled (accessories are optional).
7. No time estimate input (accessories skip time validation).
8. No seamstress assignment dropdown (accessories filtered from AssignmentStep per Plan 03).
