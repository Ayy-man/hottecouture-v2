# Phase 09-03: Mobile Cards and Full-Screen Modals

## Goal
Convert table views to cards on mobile and make modals full-screen on small devices.

## Requirements Covered
- RES-05: Convert tables to cards on mobile
- RES-06: Full-screen modals on mobile

## Current State
- `order-list-view.tsx` uses `<table>` layout
- Modals use `max-w-*` classes but not full-screen on mobile
- Modal component has basic responsive padding

## Implementation

### Task 1: Update order-list-view.tsx for mobile cards
**File:** `src/components/board/order-list-view.tsx`

Add responsive layout that shows:
- Desktop: Table view (existing)
- Mobile: Card-based list view

```tsx
// Desktop table - hidden on mobile
<div className="hidden md:block">
  <table>...</table>
</div>

// Mobile cards - hidden on desktop
<div className="md:hidden space-y-3">
  {orders.map(order => (
    <div key={order.id} className="bg-white rounded-lg border p-4">
      <div className="flex justify-between items-start mb-2">
        <span className="font-semibold">#{order.order_number}</span>
        <Badge>{order.status}</Badge>
      </div>
      <p className="text-sm text-muted-foreground">{order.client_name}</p>
      <div className="flex justify-between mt-2 text-sm">
        <span>Due: {formatDate(order.due_date)}</span>
        <span>{formatCurrency(order.total_cents)}</span>
      </div>
    </div>
  ))}
</div>
```

### Task 2: Update Modal component for full-screen on mobile
**File:** `src/components/ui/modal.tsx`

Add full-screen variant for mobile:
```tsx
<div className={cn(
  "relative w-full bg-card shadow-modal",
  // Full screen on mobile, centered modal on desktop
  "h-full md:h-auto md:max-h-[90vh]",
  "rounded-none md:rounded-lg",
  "mx-0 md:mx-4",
  sizeClasses[size],
  className
)}>
```

### Task 3: Update order-detail-modal.tsx
**File:** `src/components/board/order-detail-modal.tsx`

- Add full-screen behavior on mobile
- Add sticky header with close button
- Ensure content scrolls properly

### Task 4: Update task-management-modal.tsx
**File:** `src/components/tasks/task-management-modal.tsx`

- Same full-screen treatment as order-detail-modal
- Sticky footer with action buttons

## Verification
1. Order list shows cards on mobile, table on desktop
2. Modals fill screen on mobile (no border radius, no margin)
3. Modal content scrolls independently
4. Close button always accessible
5. No horizontal scrolling anywhere
