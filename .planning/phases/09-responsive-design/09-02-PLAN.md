# Phase 09-02: Touch-Friendly UI and Form Stacking

## Goal
Ensure all interactive elements are touch-friendly and forms stack vertically on mobile.

## Requirements Covered
- RES-03: Touch-friendly buttons (min 44px height)
- RES-04: Stack form fields vertically on mobile

## Current State
- Some buttons are already touch-friendly
- globals.css has `.touch-manipulation` and some 44px rules
- Form grids use `grid-cols-2` without mobile stacking

## Implementation

### Task 1: Add touch-friendly utility classes
**File:** `src/app/globals.css`

```css
/* Touch-friendly minimum sizes */
@layer utilities {
  .touch-target {
    min-height: 44px;
    min-width: 44px;
  }

  .touch-target-sm {
    min-height: 36px;
    min-width: 36px;
  }
}

/* Ensure all buttons meet touch targets on mobile */
@media (max-width: 767px) {
  button,
  [role="button"],
  input[type="submit"],
  input[type="button"],
  .btn {
    min-height: 44px;
  }

  input, select, textarea {
    min-height: 44px;
    font-size: 16px; /* Prevents iOS zoom on focus */
  }
}
```
