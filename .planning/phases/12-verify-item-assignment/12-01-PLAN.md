# Phase 12 Plan 01: Verify & Fix Item-Level Assignment in Production

<objective>
Client reports item-level assignment is "still broken" in Feb 3 demo - orders still assigning to one person. Investigate whether the implementation from Phase 1 is actually working in production, identify the specific failure points, and fix them.
</objective>

## Context

- Phase 1 implemented item-level assignment (4 plans, all marked complete Jan 19-20)
- Client's consulting team says: "Still assigning entire orders to one person in demo"
- This could be: a deployment issue, a UI bug where the per-item dropdowns aren't visible, or a data flow issue where assignments aren't saving per-item

## Investigation Tasks

### Task 1: Verify Database Schema in Production
- Check that `garment_service.assigned_seamstress_id` column exists in Supabase
- Check that migration 0031 and 0032 ran successfully
- Query: `SELECT column_name FROM information_schema.columns WHERE table_name = 'garment_service' AND column_name = 'assigned_seamstress_id'`

### Task 2: Verify Assignment API Route Works
- File: `src/app/api/garment-service/[id]/assign/route.ts`
- Test: PATCH request with a valid garment_service ID and user_id
- Confirm it saves to the database correctly

### Task 3: Verify Intake Flow Saves Per-Item Assignment
- File: `src/app/intake/page.tsx`
- File: `src/components/intake/` (look for AssignmentStep or GarmentServicesStep)
- Check that when creating an order, each item's `assigned_seamstress_id` is sent to the API
- Check `/api/intake/route.ts` - does it save `assigned_seamstress_id` per garment_service?

### Task 4: Verify Board/Workload Shows Per-Item Assignments
- File: `src/app/board/page.tsx`
- File: `src/lib/hooks/useBoardData.ts` or similar
- Check that filtering by seamstress filters by `garment_service.assigned_seamstress_id` (not `order.assigned_to`)
- Check workload page groups items by individual assignment

### Task 5: Fix Any Issues Found
- Apply fixes based on investigation
- Test the full flow: create order -> assign items to different people -> verify each person sees only their items
